# Agentics Vault

Welcome to the **Agentics Vault** — a structured, markdown-first knowledge base designed to bridge human readability and automated agent reliability.

This repository serves as a strict ontology-driven Wiki. It functions as a system of record that separates raw **sources**, abstract **concepts**, durable **entities**, and atomic **claims** and **evidence**.

## Getting Started

To learn about how this vault operates, please consult the core documentation:

- [**`index.md`**](index.md): The main entry point and navigation.
- [**`WIKI.md`**](WIKI.md): The human-readable schema, folder meanings, page types, and editorial guidelines.
- [**`AGENTS.md`**](AGENTS.md): The rigid behavior contract for AI agents reading or writing to this vault.
- [**`AGENT-WIKI-SPEC-v1.md`**](AGENT-WIKI-SPEC-v1.md): The full technical V1 specification.

## Intake Queue

New data or unstructured notes should first land in the [`_inbox/`](_inbox/) folder as pointer files. See [INBOX.md](INBOX.md) for details on the triage and processing lifecycle.

## Compilation Pipeline

The machine-facing caches and summary reports are not meant to be hand-edited. They are regenerated by running the compilation pipeline:

```bash
python3 _wiki/skills/compile-wiki/scripts/compile.py
```

Running this script parses the markdown and YAML frontmatter across the vault, producing structured graph elements in `_wiki/cache/` (such as `pages.json`, `claims.jsonl`, and `relations.jsonl`). This makes the underlying knowledge graph instantly digestible by agents and RAG pipelines.
